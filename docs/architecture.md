# System Architecture: AMIS Roadmap Extension

## 1. Architectural Philosophy
The AMIS Roadmap Extension is built on a **Zero-Server, Client-Side-Only** architecture. 

In traditional academic tools, user data is forwarded to a centralized backend for processing. To prioritize system resilience and strict data privacy, this extension shifts the entire computation payload to the client. All algorithms—from Grade Weighted Average (GWA) calculations to complex prerequisite pathfinding—are executed natively within the Chrome V8 engine.

## 2. Core Modules & Data Flow

The system is separated into three highly decoupled modules:

### A. Service Layer (`src/services/amisApi.js`)
Responsible for secure authentication and data extraction.
* **Token Scraping Strategy:** Utilizes a multi-phase scraping algorithm via `chrome.scripting.executeScript` to locate opaque auth tokens (`Bearer`) from local/session storage.
* **Volatile Storage:** Tokens are immediately moved to `chrome.storage.session`, ensuring they are cleared from memory the moment the browser closes.
* **Error Handling:** Implements standardized HTTP interceptors to handle `401 Unauthorized` responses gracefully, purging stale local state and prompting re-authentication.

### B. Core Logic Engine (`src/engine/CurriculumEngine.js`)
The mathematical core of the extension, functioning independently of the UI.
* **Graph Construction:** Parses raw JSON curriculum data to map courses into a **Directed Acyclic Graph (DAG)**.
* **Dependency Resolution:** Evaluates deep prerequisite chains using a custom parsing engine that handles complex `AND/OR` string conditions (e.g., `(CMSC 11 AND CMSC 12) OR COI`). 
* **Equivalence & Overrides:** Dynamically resolves course equivalence (e.g., matching `HIST 1` to `KAS 1`) to prevent double-counting units across elective and core GE requirements.
* **Optimal Path Generation:** Calculates "Dependency Weights" to prioritize bottleneck courses. It allocates subjects into strict semester loads (handling Midyear edge cases and unit overloading parameters) to project the fastest route to graduation.

### C. Presentation Layer (`src/popup/`)
A lightweight, reactive UI bound to the state generated by the Logic Engine.
* **State Management:** Maintains a centralized `state` object holding the parsed curriculum map, active program ID, and the calculated roadmap.
* **Dynamic Rendering:** Injects the optimal path into the DOM, conditionally rendering dropdowns based on the student's last completed term (e.g., automatically locking the next term to "Midyear" if the 2nd Semester just concluded).

## 3. Security Considerations
* **No Telemetry:** The application contains zero outbound network requests other than to the official `api-amis` endpoint. 
* **Content Security Policy (CSP):** Adheres strictly to Manifest V3 CSP constraints. No external scripts are evaluated, and `eval()` is strictly prohibited across the codebase.
